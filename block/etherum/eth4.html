Chapter : state machine

Ethereum = "distributed state machine" Une state machine (machine à états) est un système qui a un état à un moment donné et qui peut changer d’état lorsqu’un événement se produit. Par exemple, une porte automatique peut être ouverte ou fermée : c’est son état. Appuyer sur le bouton change l’état. Ethereum est une state machine distribuée : “Distribuée” signifie que tout le réseau possède une copie de l’état. Chaque fois qu’un changement se produit (par exemple, un transfert de crypto), tout le réseau s’accorde pour mettre à jour son état de manière consistante. 2️⃣ Ethereum vs Bitcoin Bitcoin : suit seulement l’état de la monnaie (qui possède combien de BTC). Ethereum : suit l’état de n’importe quel type de donnée (et pas seulement l’argent). 3️⃣ Clé–valeur comme mémoire Ethereum utilise une data store clé–valeur : Chaque valeur est accessible par une clé unique. Exemple : clé = "Book Title", valeur = "Mastering Ethereum". Cela ressemble au fonctionnement de la RAM d’un ordinateur : tu as une adresse (clé) et tu stockes quelque chose (valeur). 4️⃣ Ethereum = ordinateur mondial Ethereum peut charger et exécuter du code (smart contracts) dans sa “machine à états”. Le résultat de ce code modifie l’état, et la blockchain enregistre ces changements. Comme un ordinateur classique, mais le code et les changements d’état sont visibles et partagés par tout le réseau. 5️⃣ Différences clés par rapport à un PC normal Règles de consensus : Les changements d’état ne sont valides que si le réseau entier est d’accord. État distribué globalement : Chaque noeud possède la même mémoire, contrairement à un PC normal qui a sa RAM privée. 6️⃣ En résumé simple Ethereum est comme un ordinateur géant partagé dans le monde entier, où : Les données sont stockées comme dans la RAM (clé–valeur). Le code peut être exécuté pour changer l’état (smart contracts). Tout le monde est d’accord sur l’état grâce à la blockchain. Cela permet de “programmer” un ordinateur mondial fonctionnant sous consensus
