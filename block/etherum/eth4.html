Chapter 3 : distributed state machine

Definition

A state machine is a system that has a state at a given moment and can change states when an event occurs.

Example 1 an automatic door can be open or closed: that is its state. Pressing the button changes the state.

Ethereum is a distributed state machine: “Distributed” means that the entire network has a copy of the state. Every time a change occurs (for example, a crypto transfer), the entire network agrees to update its state consistently.

Ethereum vs Bitcoin

Bitcoin : suit seulement l’état de la monnaie (qui possède combien de BTC). Ethereum : suit l’état de n’importe quel type de donnée (et pas seulement l’argent).

Clé–valeur comme mémoire

Ethereum utilise une data store clé–valeur : Chaque valeur est accessible par une clé unique. Exemple : clé = "Book Title", valeur = "Mastering Ethereum". Cela ressemble au fonctionnement de la RAM d’un ordinateur : tu as une adresse (clé) et tu stockes quelque chose (valeur).

Ethereum = ordinateur mondial

Ethereum peut charger et exécuter du code (smart contracts) dans sa “machine à états”. Le résultat de ce code modifie l’état, et la blockchain enregistre ces changements. Comme un ordinateur classique, mais le code et les changements d’état sont visibles et partagés par tout le réseau.

Différences clés par rapport à un PC normal

Règles de consensus : Les changements d’état ne sont valides que si le réseau entier est d’accord. État distribué globalement : Chaque noeud possède la même mémoire, contrairement à un PC normal qui a sa RAM privée.

Resume

Ethereum est comme un ordinateur géant partagé dans le monde entier, où : Les données sont stockées comme dans la RAM (clé–valeur). Le code peut être exécuté pour changer l’état (smart contracts). Tout le monde est d’accord sur l’état grâce à la blockchain. Cela permet de “programmer” un ordinateur mondial fonctionnant sous consensus
